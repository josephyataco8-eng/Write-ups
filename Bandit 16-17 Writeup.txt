Bandit 16-17 Writeup

First, we scan the ports in the range 31000–32000 using the following nmap command:

nmap -sT -p 31000-32000

Options:

-sT: TCP scan technique. Why? There are many scanning techniques, but we choose this 
one because we do not have privileges in this laboratory, and -sT allows us to perform 
a scan without special privileges.

-p: specify the port range.

Output (relevant part only):

Not shown: 996 closed tcp ports (conn-refused)
PORT      STATE SERVICE
31046/tcp open  unknown
31518/tcp open  unknown
31691/tcp open  unknown
31790/tcp open  unknown
31960/tcp open  unknown

Since we now know which ports are open, we use the following command:

nmap -sV -p 31046,31518,31691,31790,31960 localhost

Options:

-sV: version/service detection scan. Why? It allows us to identify the services 
and see which ones offer SSL (Secure Sockets Layer). In addition, it does not 
require root privileges.

-p: specify the ports.

Output (relevant part only):

PORT      STATE SERVICE     VERSION
31046/tcp open  echo
31518/tcp open  ssl/echo
31691/tcp open  echo
31790/tcp open  ssl/unknown
31960/tcp open  echo

We now know that ports 31518 and 31790 provide SSL services, so we try to 
connect to the service using the following commands:

1. printf "password\n" | openssl s_client -connect localhost:31518 -quiet

2. printf "password\n" | openssl s_client -connect localhost:31790 -quiet

printf: prints text.

"|": passes the output of the previous command to the next one.

s_client: an OpenSSL mode that tells OpenSSL to act as a client.

1. It does not return any credential.

2. It returns an encrypted RSA private key:

-----BEGIN RSA PRIVATE KEY-----
...
-----END RSA PRIVATE KEY-----

This private key will allow us to prove our identity without providing a password.

It is required that the file where we store our credential has only read and write
permissions for us (600). Otherwise, someone else could use our credentials. Assuming 
that the file where we store the credential is called "credenciales_RSA".

we use the following command:

chmod 600 credenciales_RSA

We use the following command to log in:

ssh bandit17@bandit.labs.overthewire.org -i credenciales_RSA -p 2220

Options:

-p: specify the port.

-i: select an identity file.

And that’s it — we are now logged into bandit17.


bandit17@bandit:~$
